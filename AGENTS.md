# AGENTS.md

## Literator Source-First Documentation

This project uses Literator to generate Markdown walkthroughs from marked TypeScript source files.

The `.ts` and `.tsx` files are the source of truth. Generated `.literated.md` files should usually not be edited directly. Edit the source file, then regenerate.

## Core Rules

- Keep TypeScript source files valid TypeScript.
- Only literate files that contain `// @literator-literate`.
- Treat `//` line comments and standalone `/* ... */` block comments as Markdown prose.
- Treat normal TypeScript code as code in the generated Markdown.
- Do not emit Literator control annotations as visible Markdown prose.
- Prefer clear walkthrough prose over API-reference prose unless the project specifically asks for API docs.
- Keep the tool intentionally simple: no config files, no AST parsing, no custom output directories.

## Output Files

Generated Markdown is saved beside the source file:

```text
src/index.ts -> src/index.ts.literated.md
src/Button.tsx -> src/Button.tsx.literated.md
```

Generated `.literated.md` files are output, not source input. Do not parse them, and do not hand-edit them unless the user explicitly asks.

Generated Markdown starts with:

```md
<!-- Generated by Literator. Edit the source file instead. -->
```

Literator scans `.ts` and `.tsx` files. It skips generated Markdown and declaration files.

## Writing Literate Source

Use the opt-in annotation near the top of a file:

```ts
// @literator-literate
```

Write longer prose in standalone block comments when it improves readability:

```ts
/*
# Pancake order

This tiny file walks through a mighty little pancake orchestrator:

1. Get the recipe.
2. Make pancakes.
3. Say it is done.
*/

function getRecipe(): string {
  return "pancake recipe";
}
```

Line comments are also fine for short notes:

```ts
// This prose becomes Markdown too.
```

Mermaid diagrams and images work because comments become Markdown:

```ts
// ```mermaid
// flowchart LR
//   Recipe[Get recipe] --> Pancakes[Make pancakes]
//   Pancakes --> Done([Done])
// ```
//
// ![Diagram](./images/diagram.png)
```

## Collapse Annotations

Use collapse annotations for asides or details that should be expandable in the generated Markdown:

```ts
// @literator-collapse-start A tiny kitchen shortcut
// A real app might ask the cooking wizard for fresh instructions.
// Here, we keep a pocket-sized version in the pantry.
// @literator-collapse-end
```

If no title is provided, Literator uses:

```text
Expand this section
```

Collapse annotations wrap content in `<details>`; they do not remove the content.

## Commands

Build the CLI:

```bash
npm run build
```

Generate docs from the default `src` folder:

```bash
npx literator
```

Generate docs from another folder:

```bash
npx literator examples/src
```

When changing example source files, regenerate the checked-in output:

```bash
node dist/cli.js examples/src
```
